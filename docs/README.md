# 문서 색인

이 프로젝트는 문서 단위로 기능을 확정한 뒤 구현한다.

## 현재 문서

- [MVP 설계](./mvp-design.md): 제품 목적, DSL, 아키텍처, MVP 범위
- [기능 세분화](./feature-breakdown.md): 구현 기능을 모듈 단위로 분해한 문서
- [참조 프로젝트 분석](./reference-projects.md): 기존 프로젝트에서 가져올 개념과 피해야 할 결합 정리
- [문서 기반 작업 계획](./work-plan.md): 문서 단위 구현 순서와 완료 기준

## 작업 원칙

1. 먼저 문서에서 책임과 입출력을 확정한다.
2. 구현은 문서의 기능 단위를 기준으로 진행한다.
3. 한 번에 여러 계층을 섞지 않는다.
4. MVP에서는 CLI/compiler core를 먼저 만들고 UI는 후속 단계로 둔다.
5. 참조 프로젝트 코드는 그대로 복사하지 않고, 필요한 개념만 TypeScript 구조로 재구성한다.
